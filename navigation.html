MIConvexHull

A .Net fast convex hull library for 2, 3, and higher dimensions

Download this project as a .zip file Download this project as a tar.gz file

There are a total of seven projects within the downloadable solution: six are example applications to show you how MIConvexHull is used, and the final one is the MIConvexHull project (which compiles as MIConvexHullPlugin.dll).

As you can see there are three items here, the MIConvexHull.sln file will open in Visual Studio 2010 or later, and the main C# files for this are located in the MIConvexHull folder. Resources simply uses 2 binaries: one is the StarMath matrix library, and the other is the HelixToolkit. The latter is only needed by one of the example applications, so the only dependence for the plugin is the StarMath file (StarMath.dll). The Examples directory contains six illustrative examples for using MIConvexHull.

3DConvexHullWPF: Using Petzold’s 3D library, this example tests the ability of MIConvexHull on 3D problems. One can choose random uniformly distributed points or points on a sphere. All point on a sphere are in the convex hull, so this choice typically takes longer to run. A simple face class is created in this example as well as a vertex class (like that from the 1st example).

 

    using MIConvexHull;

    /// <summary>

    /// A vertex is a simple class that stores the position of a point, node or vertex.

    /// </summary>

    public class face : IFaceConvHull

    {

        public face()

        {

            vertices = new IVertexConvHull[3];

        }

        public IVertexConvHull[] vertices { get; set; }

        public double[] normal { get; set; }

    }

This very simple class (shown here in its entirety simple implements the vertices array and normal vector of IFaceConvHull. When the FindConvexHull routine is invoked in this project, the faces are returned as a result of the optional out argument.

            convexHullVertices = ConvexHull.FindConvexHull(out faces, typeof(face), 3);

 

4 DelauanayAndVoronoi: The Delaunay triangulation and the Voronoi diagrams for 2D can be found by projecting the points into three-dimensional space, and taking the convex hull. This example shows how the FindDelaunayTriangulation and FindVoronoiDiagrams functions are used. For Delaunay triangulation:

 

            faces = convexHull.FindDelaunayTriangulation(typeof(face));

Here the only thing returned is the faces. For native face types (like in this example and the 3rd example above), we can pass in the type as an optional argument so that MIConvexHull creates a list of faces in our preferred format. If no argument is provided, then the faces are of the type MIConvexHull->FaceData. For Voronoi:

 

            List<IVertexConvHull> nodes;

            List<Tuple<IVertexConvHull, IVertexConvHull>> edges;

            convexHull.FindVoronoiGraph(out nodes, out edges);

the function is “void” (nothing is returned) and so it is necessary to provide both out arguments for the Voronoi nodes and Voronoi edges. The nodes must be a List of IVertexConvHull objects while the edges takes advantages of the new Tuple class in .Net 4.0. The edges are pairs of the nodes returned in the first argument.

 

5TestEXE for MIConvexHull-6D Voronoi: This time-consuming example perhaps points out the weakness of MIConvexHull. The goal is to determine the Voronoi graph for a 6 dimensional problem of 500 vertices. The idea is that for a rigid body (perhaps a robot in C-space) in three-dimensional space, there are really 6 dimensions: x, y, z, theta-x, theta-y, and theta-z. This takes a very long time to run – on the order of one hour. But the results may be very useful in some applications.

 

6Using Helix To Open Models: I recently learned of the wonderful HelixToolkit and wanted to test MIConvexHull with any arbitrary models. Most of the pictures from the home page are created with the test. I attempted to include Delaunay and Voronoi calculations as well. These do work, but the display is not – or at least is not tractable with my graphics card. One thing learned from this test example is the need to avoid repeat vertices. If there are any duplicates, MIConvexHull will likely crash. It may be possible to include this check within MIConvexHull, but I want to keep it fast over being robust. If you are having trouble consider including a line of code to remove the duplicates as has been done here in line.

 

            verts = verts.Distinct(new samePoint()).ToList();
